Client Details

Use the Client App details to modify the Client App's metadata, policies and contracts. There is no need to follow the tabs in order, but you will need to fill out a minimum amount of data before the Client App can be registered with the gateway. In particular, the Client App must have at least one API contract (see tab Contracts).

Contracts

In the Contracts tab you can manage all API contracts for this Client App. An API contract is a link between this Client App and a provided API the Client App consumes.

Policies

The Policies tab allows you to manage the client-level policies that should be applied whenever a request is made to any API consumed by this Client App.

APIs

APIs

The APIs tab lists all APIs consumed by this Client App. Use the list to display also the API Key and live endpoint for each API. Use this tab to figure out how to invoke an API once you have created a contract for it.

Metrics

Metrics

The Metrics tab contains basic analytic information for the Client App. Use this tab to monitor the Client App's activity.

Change Log

The Change Log tab shows a history of all the changes made to the Client App. Essentially this is an audit log.
